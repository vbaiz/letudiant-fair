Système de scoring — Mathématiques

A. Intent Score Global — 0 à 100

Score d'engagement de l'étudiant toutes écoles confondues. Recalculé après chaque action significative.

Niveau scolaire renseigné
+20
Email réel fourni
+10
Scans stand (max 3×)
+10/scan
Swipes droite (max 2×)
+15/swipe
RDV réservé
+25
Conférence assistée
+15
Seuils
0  – 30  →  low    (exploring)
31 – 65  →  medium (comparing)
66 – 100 →  high   (deciding)
Pondération temporelle (blend booth)
age_days = NOW() - booth_capture.captured_at
decay    = max(0.5,  1.0 - (age_days / 60))

score_final = SI activité_app_existe:
                0.70 × app_intent_score + 0.30 × (booth_score × decay)
              SINON:
                booth_score × decay

B. Lead Score Par étudiant × école — 0 à 100

Qualifie l'affinité d'un étudiant pour une école spécifique. Utilisé dans l'export CRM des exposants.

Formule complète
lead_score = 0.50 × score_déclaratif + 0.50 × score_comportemental

score_déclaratif =
    0.40 × overlap_domaines(education_branches, target_fields)
  + 0.30 × match_niveau(education_level, target_levels)
  + 0.30 × proxy_orientation(education_level)

    où:
      overlap_domaines = |A ∩ B| / max(|A|, |B|)
      match_niveau     = 1.0 si correspondance, sinon 0.0
      proxy_orientation:
        seconde   → 0.30
        première  → 0.50
        terminale → 0.80
        post-bac  → 1.00

score_comportemental =
    0.35 × rdv_reservé
  + 0.25 × swipe_droite
  + 0.25 × stand_visité
  + 0.15 × conférence_assistée

Tiers : 0–40 = exploring  |  41–65 = comparing  |  66–100 = deciding

C. Score Recommandation École Score booth — 0 à 100

Calculé au score booth pour proposer les 3 meilleures écoles à un étudiant non inscrit.

Formule composite
fit_score =
    0.35 × education_level_fit      (niveau dans school.target_levels)
  + 0.25 × domain_relevance         (Jaccard: branches vs programme_domains)
  + 0.20 × academic_feasibility     (niveau booth vs minimum requis école)
  + 0.10 × specialization_alignment (bac_specialty vs préférence école)
  + 0.10 × preference_signal        (étudiant a cité cette école explicitement)
  × 100

D. Score d'Alignement Intérêt Par dossier — 0.0 à 1.0

Mesure à la création du dossier si l'école choisie correspond aux intérêts déclarés de l'étudiant.

Similarité de Jaccard
alignment = |student_fields ∩ school_fields| / |student_fields ∪ school_fields|

Exemple:
  student_fields = ["Business", "Marketing", "Finance"]
  school_fields  = ["Business", "Finance", "Management"]
  intersection   = {"Business", "Finance"}  → 2
  union          = {"Business", "Marketing", "Finance", "Management"} → 4
  alignment      = 2 / 4 = 0.50  → "Partiellement aligné"

Seuils:
  0.00 – 0.39  →  Peu aligné       (rouge)
  0.40 – 0.69  →  Partiellement    (jaune)
  0.70 – 1.00  →  Très aligné      (vert)

Les 6 clusters comportementaux

C1
Le Candidat Décidé
Cluster premium — cible prioritaire des exposants
intent_level = 'high'
dossier_count ≥ 2
avg_alignment_score ≥ 0.70
has_appointment = true
dwell_minutes > 90
Valeur business : Top leads pour les écoles. Cohorte de suivi pipeline. Preuve ROI du salon.
→ Email de nurture école, suivi RDV, tracker les résultats
C2
L'Explorateur
Large intérêt, aucune décision
intent_level = 'low'
swipe_right_count ≥ 3
dossier_count = 0
dwell_minutes 30–90
education_branches.length ≥ 3
Valeur business : Candidat au reciblage. Audience retargeting. Besoin de curation personnalisée.
→ Recommandations d'écoles personnalisées post-salon
C3
Le Désaligné
Intérêts ≠ écoles choisies
dossier_count ≥ 1
avg_alignment_score < 0.30
(domaines déclarés ≠ domaines école)
Valeur business : Valide/invalide la qualité des données profil. Signal d'exploration ou de données floues.
→ Nudge "Est-ce le bon choix ?" + alternatives alignées
C4
Le Groupe Scolaire
Visite passive, cycle long
group_id IS NOT NULL
is_minor = true OU level IN ('seconde', 'première')
dwell_minutes < 60
stand_scan_count ≤ 2
Valeur business : Audience future. Cycle de vente long (2–3 ans). ROI indirect via enseignant.
→ Récap post-visite à l'enseignant, nurture light
C5
Le Fantôme
Présent mais invisible
entry_scan = true
stand_scan_count = 0
dossier_count = 0
intent_level = 'low'
dwell_minutes < 30
Valeur business : Gonfle les métriques de vanité. Important à exclure des taux de conversion réels.
→ Enquête post-salon pour recapturer le signal d'intention
C6
Le Convertisseur Post-Salon
Tiède pendant, actif après
intent_level = 'medium' pendant l'événement
dossier_count ≥ 1 (créé après le salon)
application_outcome IN ('accepted', 'waitlisted')
Valeur business : Prouve le ROI salon même pour les visiteurs medium-intent. Attribution revenue.
→ Success story content, preuve d'attribution client

Requête SQL — Export matrice de clustering

SQL — PostgreSQL / Supabase
SELECT
  u.id,
  u.education_level,
  u.intent_level,
  u.last_dwell_minutes,
  COUNT(DISTINCT s.id) FILTER (WHERE s.channel = 'stand')        AS stand_scans,
  COUNT(DISTINCT m.id) FILTER (WHERE m.student_swipe = 'right')  AS swipe_rights,
  COUNT(DISTINCT a.id)                                            AS appointments,
  COUNT(DISTINCT d.id)                                            AS dossiers,
  ROUND(AVG(d.interest_alignment_score)::numeric, 2)             AS avg_alignment,
  BOOL_OR(d.application_outcome = 'accepted')                    AS got_accepted,
  u.group_id IS NOT NULL                                         AS is_group_visitor,
  u.is_minor
FROM users u
LEFT JOIN scans s
  ON s.user_id = u.id
LEFT JOIN matches m
  ON m.student_id = u.id
LEFT JOIN appointments a
  ON a.student_id = u.id AND a.status != 'cancelled'
LEFT JOIN application_dossiers d
  ON d.user_id = u.id
GROUP BY u.id;

→ Exporter en CSV, importer dans Python/sklearn, appliquer K-Means ou DBSCAN sur les features normalisées.

Requête : Temps par étape de candidature

SQL — dossier_stage_events
SELECT
  dossier_id,
  from_status,
  to_status,
  ROUND(
    EXTRACT(EPOCH FROM (
      LEAD(occurred_at) OVER (PARTITION BY dossier_id ORDER BY occurred_at)
      - occurred_at
    )) / 86400, 1
  ) AS days_in_stage
FROM dossier_stage_events;

Requête : Taux de conversion stand → dossier par école

SQL — scans JOIN application_dossiers
SELECT
  sch.name                                          AS school_name,
  COUNT(DISTINCT s.user_id)                         AS stand_visitors,
  COUNT(DISTINCT d.user_id)                         AS dossier_creators,
  ROUND(
    100.0 * COUNT(DISTINCT d.user_id)
    / NULLIF(COUNT(DISTINCT s.user_id), 0), 1
  )                                                  AS conversion_pct
FROM scans s
JOIN stands st     ON st.id = s.stand_id
JOIN schools sch   ON sch.id = st.school_id
LEFT JOIN application_dossiers d
  ON d.school_id = st.school_id AND d.user_id = s.user_id
WHERE s.channel = 'stand'
GROUP BY sch.id, sch.name
ORDER BY conversion_pct DESC;

Questions business répondables

ROI salon post-événement Quel % des étudiants high-intent au salon ont créé un dossier post-salon ?
users (intent_level='high') JOIN application_dossiers
Alignement vs succès Les étudiants avec alignment_score > 0.70 ont-ils un meilleur taux d'acceptation ?
GROUP BY alignment_bucket, AVG(outcome='accepted')
Goulots d'étranglement Où les candidatures stagnent-elles ? (jours moyens par étape)
dossier_stage_events → time-in-stage query
Désalignement par école Quelles écoles reçoivent le plus de candidatures désalignées ?
GROUP BY school_id WHERE alignment_score < 0.30
Conversion stand → dossier Quel est le taux stand_scan → dossier_créé par école ?
scans JOIN application_dossiers ON school_id
Domaines les plus actifs Quels education_branches ont le plus fort taux de candidature post-salon ?
unnest(education_branches) → application_dossiers count
Offre vs demande Y a-t-il des domaines sur-demandés par rapport à l'offre des écoles présentes ?
v_field_interest JOIN schools.target_fields
Candidats à nurture Quels étudiants (cluster Explorateur) méritent une recommandation post-salon ?
intent='low', swipe_rights≥3, dossier_count=0
Temps de décision Quel est le délai moyen de décision par école (dossier créé → outcome) ?
outcome_received_at - created_at GROUP BY school_id
Complétude dossier vs entretien Les étudiants qui uploadent 4/4 documents ont-ils plus d'entretiens ?
application_documents COUNT JOIN dossier_stage_events WHERE to_status='interview'
Qualité leads exportés Les écoles qui exportent leurs leads ont-elles un meilleur taux de conversion ?
leads WHERE exported_at IS NOT NULL JOIN appointments.status='attended'
Attribution revenue Quel % d'acceptations peut être attribué directement au salon (cluster C6) ?
cluster C6 → application_outcome='accepted' COUNT / total_accepted
