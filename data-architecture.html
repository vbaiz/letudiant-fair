What each role can read, write, or only see in aggregate (enforced by RLS)

Registered fair attendee. Auth via Supabase. May be minor (guardian required).
Own user profile R/W
School catalogue Read
School reels + articles Read
Own scans / intent score Read
Own matches (swipes) R/W
Own appointments R/W
Saved articles + reels R/W
Own group membership Read
Lead records (own) Hidden
Other students' data No access

Exhibitor (School)

School staff managing their booth, reels, and reviewing interest data.
Own school profile R/W
Own formations R/W
Own school reels R/W

Appointments (count only) Aggregate only
Export leads (post-fair) Write (export)

Teacher

Creates and manages student groups for class trips to the fair.
Own group R/W
Group member list Read
Invite link generation Write
Student profiles (own group) Aggregate only

Parent / Guardian

Required for minor students. Provides consent & oversight.

Full access via service-role key. Bypasses all RLS. Platform operators.

Scoring triggers Write
DB seeding / repair Write
Analytics aggregates Read
GDPR data (all) Full access

Scoring Algorithms

Three independent scoring engines — all in lib/scoring/

Intent Score

lib/scoring/intentScore.ts

Measures how engaged a student is at the fair. Updated after each behavioral signal.

Education level filled20 pts
Real email (not temp)10 pts
Stand QR scans (capped 30)10 pts ea.
Swipes right (capped 30)15 pts ea.
Appointment booked25 pts
Conference attended15 pts (once)
Output: score 0–140
Levels: low (0–30) · medium (31–65) · high (66+)
Stored on: users.intent_score, users.intent_level

Lead Score

lib/scoring/leadScore.ts

Qualifies a student as a lead for a specific school. Run post-event via /api/leads/compute.

Declarative 50%
Field/branch overlap with school~30 pts
Education level match~20 pts
Behavioral 50%
Appointment booked~20 pts
Swiped right on school~15 pts
Visited school stand~10 pts
Attended school conference~5 pts
Output: score_value 0–100
Tiers: exploring · comparing · deciding
Stored on: leads.score_value, leads.score_tier

School Recommendation

lib/scoring/schoolRecommendation.ts

Used by the phygital score booth. Takes academic grade (0–20) + student profile and returns top-3 schools.

Domain relevancehigh weight
Academic feasibility (grade vs. bac counts)medium
Specialization alignmentmedium
Student declared preferenceslow
Output: top-3 SchoolRow[] with fit_score 0–100
Tiers: strong · moderate · exploratory
Called by: POST /api/booth/recommendations

School Ranking (Discover)

lib/supabase/schoolRanking.ts

Personalizes the school listing order for logged-in students in the Discover tab.

Field/branch overlap60%
Education level match20%
Region proximity20%
Behavioral boost (prior swipes)bonus
Output: sorted SchoolRow[]
Used by: Discover page · client-side, no API
Also mirrors: programRanking.ts for formations
