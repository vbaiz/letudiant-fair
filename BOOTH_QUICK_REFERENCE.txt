📋 WHAT WAS BUILT
─────────────────────────────────────────────────────────────────────────────
✅ Database (Supabase)
   - Table: booth_captures (staff-entered orientation data for non-app users)
   - Auto-merge trigger (student signup → booth data syncs to profile)
   - RLS policies (exhibitors see only their stand captures)

✅ API Endpoint
   - POST /api/booth/capture-orientation (service role, no auth)
   - Accepts: orientation_stage, education_level, education_branches, contact info
   - Returns: {success, capture_id, message}

✅ Form Component
   - 3-question form (~30 seconds to fill)
   - Q1: How far along? (Exploring/Comparing/Deciding)
   - Q2: Interested in what? (6 education branches)
   - Q3: Can we contact? (optional email/phone opt-in)

✅ Exhibitor Dashboard
   - New card: "Capture Booth (Non-app)" with orientation breakdown
   - Shows: Total captures, stage distribution, percentages
   - Yellow background to distinguish from app data

✅ Enhanced Scoring
   - Booth signals weight 30% if app activity exists
   - Booth signals weight 100% if no app activity
   - Recency weighting: <7 days = full weight, decays after

⚠️ IF THINGS GO WRONG
─────────────────────────────────────────────────────────────────────────────
API fails after migration:
  → Check SUPABASE_SERVICE_ROLE_KEY is set in Vercel
  → Check migration ran successfully (no errors in SQL Editor)
  → Redeploy code after fixing

Exhibitor dashboard shows no booth card:
  → This is expected if no captures exist yet
  → After first staff capture, card appears
  → Check Supabase: SELECT count(*) FROM booth_captures;

Student signup doesn't auto-merge:
  → Verify trigger exists: SELECT * FROM pg_trigger WHERE tgname='trigger_merge_booth_on_user_create';
  → Check merge function exists
  → Run manually: SELECT merge_booth_capture_to_user(user_id, email);

Complete rollback:
  DROP TABLE public.booth_captures;
  (cleans up everything, migration can be re-applied)
